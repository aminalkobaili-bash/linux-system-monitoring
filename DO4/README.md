## 1. Генерация файлов

### Требования

Создать bash-скрипт, принимающий 6 аргументов:

```bash
main.sh /path 4 az 5 az.az 3kb
```

### Параметры

1. Путь для создания директорий
2. Количество вложенных директорий
3. Символы для имён директорий (до 7)
4. Количество файлов в каждой директории
5. Символы для имён файлов и расширений
6. Размер файлов (до 100 KB)

### Условия

- Имена формируются только из заданных символов
- Каждый символ должен использоваться минимум один раз
- Длина имени ≥ 4 символов + дата в формате `DDMMYY`
- Порядок символов должен сохраняться
- Пример:

```bash
./aaaz_021121/
./aaazzzz_021121
```

### Дополнительно

- Создать структуру директорий и файлов
- Прекратить выполнение, если свободно менее 1 ГБ
- Вести лог:
  - полный путь
  - дата создания
  - размер файлов

## 2. Заполнение файловой системы

### Требования

Скрипт принимает 3 аргумента:

```bash
main.sh az az.az 3Mb
```

### Параметры

1. Символы для директорий
2. Символы для файлов
3. Размер файлов (до 100 MB)

### Условия

- Имена формируются аналогично предыдущему пункту
- Длина имени ≥ 5 символов + дата
- Создание происходит в разных местах системы (кроме путей с `bin` и `sbin`)
- Количество директорий — до 100
- Количество файлов — случайное

### Контроль ресурсов

```bash
df -h /
```

- Завершить работу при остатке < 1 ГБ

### Логирование

- Путь, дата, размер
- Время начала и окончания
- Общее время выполнения

## 3. Очистка файловой системы

### Требования

Скрипт принимает 1 параметр (режим работы):

- 1 — удаление по лог-файлу
- 2 — удаление по времени
- 3 — удаление по шаблону имени

### Особенности

- При удалении по времени:
  - задаётся диапазон (с точностью до минуты)
  - удаляются все файлы, попадающие в интервал

## 4. Генерация логов

### Требования

Создать скрипт (или программу), генерирующий 5 лог-файлов nginx.

### Условия

- Формат: **combined**
- Каждый файл = один день
- Количество записей: 100–1000

### Генерируемые поля

- IP-адрес
- Код ответа:
  - 200, 201, 400, 401, 403, 404, 500, 501, 502, 503
- HTTP-метод:
  - GET, POST, PUT, PATCH, DELETE
- Дата (возрастающая)
- URL
- User-Agent

### Дополнительно

- В комментариях указать значение кодов ответа

## 5. Анализ логов

### Требования

Скрипт принимает 1 параметр (1–4):

1. Сортировка по коду ответа
2. Уникальные IP
3. Запросы с ошибками (4xx, 5xx)
4. Уникальные IP с ошибками

Использовать:

```bash
awk
```

## 7. Мониторинг системы (Prometheus + Grafana)

### Требования

- Установить Prometheus и Grafana
- Обеспечить доступ к веб-интерфейсам

### Дашборд должен содержать:

- загрузку CPU
- использование RAM
- свободное место на диске
- операции ввода/вывода

### Тестирование

- Запустить скрипт заполнения файловой системы
- Наблюдать за дисковой нагрузкой

```bash
stress -c 2 -i 1 -m 1 --vm-bytes 32M -t 10s
```

- Проверить:
  - CPU
  - RAM
  - диск

## 9. Реализация собственного экспортера

### Требования

Создать скрипт или программу, собирающую:

- CPU
- RAM
- диск

### Формат

- Генерация HTML-страницы в формате Prometheus
- Отдача через nginx

### Обновление

- циклически в скрипте  
  или
- через cron (не чаще 1 раза в 3 секунды)

### Интеграция

- Настроить Prometheus для сбора данных

### Проверка

- Выполнить нагрузочные тесты (аналогично разделу 7)
