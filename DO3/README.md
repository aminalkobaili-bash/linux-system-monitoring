## 1. Базовый скрипт

### Задача

Создать простой bash-скрипт.

### Требования

- Скрипт принимает один аргумент.
- Аргумент является строковым значением.
- Скрипт выводит переданное значение в терминал.
- Если аргумент является числом, необходимо вывести сообщение об ошибке.

## 2. Сбор информации о системе

### Задача

Разработать скрипт, выводящий ключевые параметры системы.

### Требования

Скрипт должен отображать:

- HOSTNAME — имя хоста
- TIMEZONE — временная зона (включая смещение UTC)
- USER — текущий пользователь
- OS — тип и версия ОС
- DATE — текущее время
- UPTIME — время работы системы
- UPTIME_SEC — uptime в секундах
- IP — IP-адрес интерфейса
- MASK — маска сети
- GATEWAY — адрес шлюза
- RAM_TOTAL — общий объём памяти (GB)
- RAM_USED — используемая память (GB)
- RAM_FREE — свободная память (GB)
- SPACE_ROOT — размер корневого раздела (MB)
- SPACE_ROOT_USED — занятое место (MB)
- SPACE_ROOT_FREE — свободное место (MB)

### Дополнительно

- После вывода данных запросить у пользователя подтверждение сохранения.
- Поддерживаются ответы `Y` и `y`.
- При подтверждении создаётся файл в текущей директории.
- Формат имени файла:

```bash
DD_MM_YY_HH_MM_SS.status
```

- Файл содержит тот же вывод, что и в терминале.

## 3. Цветной вывод данных

### Задача

Модифицировать скрипт из предыдущего этапа.

### Требования

- Убрать сохранение в файл.
- Добавить поддержку 4 параметров при запуске:

```bash
script.sh 1 3 4 5
```

### Параметры

- 1 — фон названий
- 2 — цвет текста названий
- 3 — фон значений
- 4 — цвет текста значений

### Цвета

- 1 — white
- 2 — red
- 3 — green
- 4 — blue
- 5 — purple
- 6 — black

### Ограничения

- Цвет текста и фона одного блока не должны совпадать.
- При некорректных параметрах:
  - вывести сообщение об ошибке
  - завершить выполнение скрипта

## 4. Конфигурация цветовой схемы

### Задача

Перенести параметры оформления в конфигурационный файл.

### Требования

- Скрипт запускается без аргументов.
- Цвета задаются в конфигурационном файле.

### Пример конфигурации

```bash
column1_background=2
column1_font_color=4
column2_background=5
column2_font_color=1
```

### Поведение

- Если параметр отсутствует — используется значение по умолчанию.
- После вывода информации необходимо вывести используемую цветовую схему.

### Пример вывода

```bash
Column 1 background = 2 (red)
Column 1 font color = 4 (blue)
Column 2 background = 5 (purple)
Column 2 font color = 1 (white)
```

### Для значений по умолчанию

```bash
Column 1 background = default (black)
Column 1 font color = default (white)
Column 2 background = default (red)
Column 2 font color = default (blue)
```

## 5. Анализ файловой системы

### Задача

Реализовать скрипт для анализа содержимого директории.

### Требования

- Скрипт принимает один аргумент — путь к директории.
- Путь должен заканчиваться символом `/`.

Пример:

```bash
script.sh /var/log/
```

### Выводимые данные

- Общее количество директорий (включая вложенные)
- Топ-5 директорий по размеру
- Общее количество файлов
- Количество файлов по типам:
  - `.conf`
  - текстовые
  - исполняемые
  - `.log`
  - архивы
  - символические ссылки
- Топ-10 файлов по размеру (с указанием типа)
- Топ-10 исполняемых файлов (с MD5)
- Время выполнения скрипта

### Формат вывода

```bash
Total number of folders (including all nested ones) = ...
TOP 5 folders of maximum size arranged in descending order:
...
Total number of files = ...
Number of:
...
TOP 10 files of maximum size arranged in descending order:
...
TOP 10 executable files of the maximum size arranged in descending order:
...
Script execution time (in seconds) = ...
```
